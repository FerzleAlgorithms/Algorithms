Minimum Coin Change

Problem Description

Given a collection of coin denominations and a target amount, determine the minimum number of coins required to make exactly that amount. If it is impossible to make the exact amount, output -1.

Input

An integer n (the number of denominations), followed by a list of n positive integers c1, c2, …, cn (the coin values), and an integer T (the target amount).

Output

A single integer: the fewest number of coins needed to form T, or -1 if no combination of coins adds up to T.

Alternatively, we also might want to know which coins to use.

Motivation/Applications

Finding the minimum number of resources needed to meet a requirement appears in budgeting, resource allocation, and making change with limited coins. It models optimization in unbounded-knapsack scenarios and appears in logistics and operations research.

Examples

Example 1
Input: n = 3, coins = [1, 2, 5], T = 11
Output: 3
Explanation: Use two 5-value coins and one 1-value coin (5 + 5 + 1 = 11).

Example 2
Input: n = 2, coins = [2, 4], T = 7
Output: -1
Explanation: It is impossible to sum to 7 using only 2- and 4-value coins.

Common Algorithms/Techniques

  • Dynamic programming (bottom-up array dp[0…T] where dp[x] is min coins for amount x)
  • Breadth-first search on implicit “amount + coin” graph
  • Greedy algorithm (only for canonical coin systems, e.g., U.S. currency)

Variants/Related Problems

  • Coin Changing (counting number of ways)
  • Subset Sum (decision problem)
  • Unbounded Knapsack (optimize total value instead of coin count)
