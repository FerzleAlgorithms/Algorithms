Edit Distance

Problem Description

Given two strings, determine the minimum number of operations required to convert one string into the other. The allowed operations are:

  • Insert a character
  • Delete a character
  • Substitute one character for another

Input

Two strings X and Y, consisting of lowercase letters.

Output

An integer representing the minimum number of operations required to transform X into Y. Optionally, the list of operations to perform the transformation.

Motivation/Applications

Edit distance has applications in computational linguistics, spell checking, DNA sequencing, plagiarism detection, and other areas where similarity between strings is important. It's also known as the Levenshtein distance.

Examples

Example 1
Input:
X = "kitten"
Y = "sitting"

Output:
3

Explanation:
kitten → sitten (substitute 'k' with 's')
sitten → sittin (substitute 'e' with 'i')
sittin → sitting (insert 'g')

Example 2
Input:
X = "flaw"
Y = "lawn"

Output:
2

Explanation:
flaw → law (delete 'f')
law → lawn (insert 'n')

Common Algorithms/Techniques

  • Dynamic Programming
  • Memoization
  • Recursive Backtracking (inefficient)

Variants/Related Problems

  • Longest Common Subsequence
  • Damerau-Levenshtein Distance (adds transposition)
  • Hamming Distance (only substitutions, strings must be same length)
