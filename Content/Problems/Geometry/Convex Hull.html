Convex Hull

Problem Description

Given a set of points in the plane, the convex hull is the smallest convex polygon that contains all the points. In other words, it is the tightest boundary that encloses all the points such that every internal angle is less than or equal to 180 degrees, and all the points lie inside or on the boundary.

Input

A list of n points in the plane, each specified by two real numbers: their x- and y-coordinates.

Output

The vertices of the convex hull in order (either clockwise or counter-clockwise), with no repeated points. The output should include only the points on the hull.

Motivation/Applications

Convex hulls are foundational in computational geometry and appear in many applications such as collision detection, shape analysis, image processing, pattern recognition, and geographical mapping. For example, the convex hull can define the boundary of a cluster of data points or help identify the outline of an object in a digital image.

Common Algorithms/Techniques

  • Graham's Scan (O(n log n))
  • Andrew's Monotone Chain Algorithm (O(n log n))
  • Jarvis March / Gift Wrapping (O(nh))
  • QuickHull (average O(n log n))
