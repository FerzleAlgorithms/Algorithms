Array Partition

Problem Description

Given an array \(A\) of \(n\) elements, select the first element \(p = A[0]\) as the pivot. Rearrange \(A\) in place so that all elements \(\le p\) precede it and all elements \(> p\) follow it, then place \(p\) into its correct position \(i\). Return \(i\), the final index of the pivot.

More generally (when not applied to an entire array), given an array \(A\) and indices \(\text{low}\) and \(\text{high}\) (with \(\text{low} < \text{high}\)), rearrange the elements in the portion of \(A\) from \(\text{low}\) through \(\text{high}\) as follows:

  1. Select a pivot element—typically the first element, \(A[\text{low}]\).
  2. Reorder the elements so that:
    • Every element \(\le p\) is positioned to its left.
    • Every element \(\ge p\) is positioned to its right.
    • The pivot element is positioned in its correct sorted location.
  3. Return the index of the pivot element.

Input

An integer array \[ A = [\,a_0, a_1, a_2, \dots, a_{n-1}\,], \] with \(n \ge 1\), and (optionally) a range \((\text{low}, \text{high})\), assumed to be \((0, n-1)\) if not specified.

Output

The array \(A\) has been modified as specified, and the index \(j\) where \(p\) resides is returned.

Motivation/Applications

This partition step is central to QuickSort and Quickselect, facilitating divide-and-conquer sorting and selection. It also supports efficient order-statistic computations and data filtering.

Examples

Example 1
Input: \(A = [5,3,8,4,2,7,1,6],\quad p = 5\)
Possible Output: \([3,4,2,1,5,8,7,6]\)
Pivot index: \(4\)

Note: Any solution is valid as long as \(\{1,2,3,4\}\) appears in the first four positions (in any order), \(5\) appears at index 4, and \(\{6,7,8\}\) occupy the last three positions (in any order).

Example 2
Input: \(A = [2,2,2,2],\quad p = 2\)
Possible Output: \([2,2,2,2]\)
Pivot index: any \(j\in\{0,1,2,3\}\)

Note: Depending on implementation, any pivot index between 0 and 3 (inclusive) is valid.

Common Algorithms/Techniques

  • Lomuto’s Partition: scan with a store index, moving elements \(\le p\) left and swapping the pivot into place.
  • Hoare’s Partition: use two pointers that converge, swapping out-of-place elements, then placing the pivot.
  • In-Place Two-Pointer: generic approach with low/high indices.
