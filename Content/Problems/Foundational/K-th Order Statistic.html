K-th Order Statistic

Problem Description

Given an unsorted array of \(n\) elements and an integer \(k\) (1 ≤ \(k\) ≤ \(n\)), find the element that would appear in the \(k\)-th position if the array were sorted in non-decreasing order.

Input

An array of \(n\) comparable elements (e.g., numbers), and an integer \(k\), where 1 ≤ \(k\) ≤ \(n\).

Output

The \(k\)-th smallest element of the array (i.e., the element of rank \(k\) in sorted order).

Motivation/Applications

Selection of order statistics is fundamental in statistics (e.g., median finding when \(k\) = ⌈\(n\)/2⌉), in database systems (e.g., finding top-\(k\) results), and in optimization (e.g., computing quantiles for robust estimators). Efficient selection avoids the cost of full sorting when only one element of a specific rank is required.

Examples

Example 1
Input: [7, 10, 4, 3, 20, 15], \(k = 3\)
Output: 7
Explanation: Sorted order is [3, 4, 7, 10, 15, 20], so the 3rd element is 7.

Example 2
Input: [12, 3, 5, 7, 4, 19, 26], \(k = 1\)
Output: 3
Explanation: Sorted order is [3, 4, 5, 7, 12, 19, 26], so the 1st element is 3.

Common Algorithms/Techniques

  • Quickselect (randomized selection, average \(O(n)\) time)
  • Median of Medians (deterministic selection, worst-case \(O(n)\) time)
  • Sort and index (\(O(n \log n)\) time, simple but not optimal for large \(n\))
  • Use a Min Heap. Build a Min Heap in \(O(n)\) time and then extract the minimum \(k\) times (each in \(O(\log n)\)), giving an \(O(n + k\log n)\) time algorithm using \(O(n)\) extra space.

Variants/Related Problems

  • Selection of the maximum, minimum, or median (\(k = 1\), \(k = n\), \(k = \lceil n/2\rceil\)
  • Top-\(k\) largest or smallest elements (returning a set instead of a single element)
  • Weighted median (order statistic with weights)
