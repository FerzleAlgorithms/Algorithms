Closest Pair

Problem Description

Given a set of points in the plane, the goal is to find the pair of points that are closest to each other, based on the Euclidean distance between them.

Input

A list of n distinct points in 2D space, where each point is represented by its Cartesian coordinates (x, y). Coordinates are typically integers or real numbers.

Output

The pair of points with the smallest Euclidean distance between them. The output may also include the value of that minimum distance.

Motivation/Applications

The Closest Pair problem arises in fields such as pattern recognition, clustering, computational geometry, and robotics. It is a fundamental subproblem in algorithms that deal with spatial proximity and can also serve as a basis for solving more complex geometric tasks.

Examples

Example 1
Input:
(1, 1), (4, 4), (8, 2), (5, 5), (7, 7), (6, 4)
Output:
Closest pair: One solution is (4, 4) and (5, 5) with distance √2 ≈ 1.41.


Example 2
Input:
(3, 7), (6, 3), (2, 5), (8, 6), (5, 6), (6, 6), (4, 1)
Output:
Closest pair: (5,6) and (6,6) with distance 1

Common Algorithms/Techniques

  • Brute Force: Compare all pairs (O(n²))
  • Divide and Conquer: Sort points and recursively merge (O(n log n))
  • Plane Sweep with Data Structures: Alternative O(n log n) solution
