Subset Sum

Problem Description

Given a set of non-negative integers and a target sum S, determine whether there exists a subset of the given integers that sums exactly to S. Return true if such a subset exists, and false otherwise.

Input

An array A of n integers (A[0..n−1]) and an integer S; Constraints: n ≥ 0, each A[i] ≥ 0, and S ≥ 0.

Output

A boolean: true if there exists indices i₁,…,iₖ with A[i₁] + … + A[iₖ] = S. Otherwise, false.

Motivation/Applications

The Subset Sum problem models resource allocation and combinatorial verification tasks: filling a knapsack to exact capacity, partitioning workloads evenly, or checking financial transactions for exact balances. It also underpins cryptographic knapsack schemes.

Examples

Example 1
Input:
A = [3, 34, 4, 12, 5, 2], S = 9
Output:
true (subset {4, 5} sums to 9)

Example 2
Input:
A = [3, 34, 4, 12, 5, 2], S = 30
Output:
false (no subset sums to exactly 30)

Common Algorithms/Techniques

  • Dynamic Programming (pseudopolynomial time DP table)
  • Backtracking / Recursive subset exploration
  • Meet-in-the-Middle (splitting into two halves)
  • Bitset or boolean-array optimizations

Variants/Related Problems

  • Partition Problem (special case with S = totalSum/2)
  • k-Sum Problem (fixed subset size)
  • Counting Subsets with Given Sum
  • 0-1 Knapsack Problem
