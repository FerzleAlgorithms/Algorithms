Travelling Salesman Problem

Problem Description

The Travelling Salesman Problem (TSP) asks: given a list of cities and the distances between each pair, what is the shortest possible route that visits each city exactly once and returns to the origin city?

Input

A complete weighted graph \( G = (V, E) \) where:

  • \( V \) is a set of cities (typically integers or labels).
  • \( E \) is a set of non-negative edge weights representing distances between each pair.

Output

A tour that visits each city once and returns to the start, along with its total cost.

Motivation/Applications

The TSP is crucial in logistics, manufacturing, chip design, and scheduling. Solutions lead to cost savings and better efficiency in delivery and service routing.

Common Algorithms/Techniques

  • Brute Force (all permutations)
  • Held-Karp Algorithm (Dynamic Programming)
  • Branch and Bound
  • Greedy Heuristics (Nearest Neighbor)
  • Christofides' Algorithm (Approximation)
  • Metaheuristics (e.g., Simulated Annealing, Genetic Algorithms)

Variants/Related Problems

  • Asymmetric TSP
