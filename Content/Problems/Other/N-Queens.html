N-Queens

Problem Description

The N-Queens problem asks whether it is possible to place \(N\) queens on an \(N \times N\) chessboard such that no two queens threaten each other. This means no two queens may share the same row, column, or diagonal. The goal is to find one such arrangement (or all such arrangements).

Input

A single integer \(N\) (typically \(N \geq 1\)), representing the size of the board and the number of queens to place.

Output

A solution is a list of \(N\) integers \(c_0, c_1, \ldots, c_{N-1}\) such that placing a queen in row \(i\) and column \(c_i\) results in a valid board configuration. Alternatively, output all such configurations if needed.

Motivation/Applications

The N-Queens problem is a classic benchmark for backtracking and constraint satisfaction. It is widely used in artificial intelligence, algorithms, and programming competitions to teach efficient pruning and recursive problem solving. It also has applications in scheduling, register allocation, and layout design.

Examples

Example 1
Input: \(N = 4\)
Output (one solution): [1, 3, 0, 2]
Board layout:
. Q . .
. . . Q
Q . . .
. . Q .

Example 2
Input: \(N = 3\)
Output: No solution exists.

Common Algorithms/Techniques

  • Exhuastive Search (very inefficient!)
  • Backtracking
  • Min-Conflicts (a local search heuristic used to solve constraint satisfaction problems)
  • Constraint Solvers
  • Genetic Algorithm

Variants/Related Problems

  • N-Rooks: Place \(N\) rooks so that no two attack each other (rows and columns only).
  • N-Queens Counting: Count the number of distinct solutions for given \(N\).
  • Knight Tour: Given a starting location, asks whether a knight can move to every square on an \(n\times n\) chessboard exactly once using legal knight moves.
  • Sudoku: Another classic constraint satisfaction problem.
