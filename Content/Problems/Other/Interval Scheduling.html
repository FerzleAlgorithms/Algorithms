Interval Scheduling

Problem Description

Given a set of intervals, each with a start time \(s_i\) and a finish time \(f_i\), select the maximum number of non-overlapping intervals. That is, choose a subset of intervals such that no two intervals overlap, and the subset size is maximized.

Input

A list of \(n\) intervals, where each interval is represented by two numbers: the start time \(s_i\) and the finish time \(f_i\). You may assume \(0\leq s_i \lt f_i\) for all \(i\), and \(n\geq 1\).

Output

A maximum-size subset of the intervals such that no two intervals in the subset overlap.

Motivation/Applications

Interval scheduling arises in many real-world applications such as CPU task scheduling, event planning, and bandwidth allocation, where resources must be scheduled without conflicts to maximize utilization.

Examples

Example 1

Input: (0,4), (1,3), (2,4), (3,6), (4,5), (5,7)

Output: (1,3), (3,6), (5,7)

Example 2

Input: (0,5), (1,4), (3,5), (3,6), (4,7), (5,8), (6,7), (7,11), (8,10), (9,11)

Output: (1,4), (4,7), (7,11)

Common Algorithms/Techniques

  • Greedy algorithm: sort intervals by increasing finish time and iteratively select
  • Dynamic programming for weighted variants

Variants/Related Problems

  • Weighted Interval Scheduling: Each interval has an associated weight; select the subset of non-overlapping intervals that maximizes the total weight. This variant is solved via dynamic programming in O(n log n) time.
  • Interval Partitioning: Assign all intervals to the minimum number of resources so that no overlapping intervals share the same resource. A greedy algorithm using a min-heap on finish times yields an optimal assignment in O(n log n) time.
  • Scheduling with Deadlines and Profits: Each job has a deadline and a profit; schedule jobs of unit length to maximize total profit before their deadlines. A greedy strategy with a priority queue achieves an optimal solution in O(n log n) time.
