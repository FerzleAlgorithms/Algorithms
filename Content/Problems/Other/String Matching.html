Exact String Matching

Problem Description

Given a pattern string \( P \) of length \( m \) and a text string \( T \) of length \( n,\) determine whether \( P \) appears in \( T,\) and if so, return the index or indices at which it occurs.

Input

A string \( P \) (the pattern) of length \( m \) and a string \( T \) (the text) of length \( n \), where \( 1 \le m \le n \).
Both strings consist of characters from a fixed alphabet, often ASCII.

Output

The list of starting indices in \( T \) where \( P \) occurs exactly. Some versions of the problem may instead return only the first index (or -1 if \( P \) does not occur).

Motivation/Applications

String matching is fundamental to a wide variety of applications, including text search (e.g., Ctrl+F), DNA sequence analysis, plagiarism detection, and search engines. Fast string search algorithms are also critical in file systems and compression utilities.

Examples

Example 1
Input:
Pattern: ana
Text: bananas

Output:
[1, 3]
Explanation: "ana" appears starting at indices 1 and 3.

Example 2
Input:
Pattern: abc
Text: abdabcbabc

Output:
[3, 7]
Explanation: "abc" appears starting at indices 3 and 7.

Common Algorithms/Techniques

  • Naive Brute Force
  • Knuth-Morris-Pratt (KMP)
  • Boyer-Moore
  • Horspool
  • Rabin-Karp (Rolling Hash)

Variants/Related Problems

  • Approximate String Matching (with errors)
  • Multiple Pattern Matching (Aho-Corasick)
  • Longest Common Substring
  • Regular Expression Matching
