Fractional Knapsack

Problem Description

Given a set of n items, each with a weight wi and a value vi, and a knapsack with capacity W, determine the maximum total value you can carry. You may take fractions of items.

Input

  • An integer n, the number of items.
  • An integer W, the capacity of the knapsack.
  • For each item i (1 ≤ i ≤ n):
    • Value vi (non-negative integer)
    • Weight wi (positive integer)

Output

A single number: the maximum total value achievable with capacity ≤ W, allowing fractional parts of items.

Motivation/Applications

The fractional knapsack models scenarios where resources can be divided, such as loading cargo of divisible goods, investing funds across opportunities, or allocating bandwidth. Its greedy solution gives insight into resource allocation and serves as a foundation for more complex variants.

Common Algorithms/Techniques

  • Greedy by sorting items in decreasing order of value-to-weight ratio: O(n log n).
  • Linear-time selection if ratios precomputed and bucketed (advanced).
