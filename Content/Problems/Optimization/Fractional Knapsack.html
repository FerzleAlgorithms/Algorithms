Fractional Knapsack

Problem Description

Given a set of \(n\) items, where each item \(i\) has weight \(w_i\) and value \(v_i\), and a knapsack capacity \(W\), choose fractions \(x_i\in[0,1]\) for each item to maximize \[ \sum_{i=1}^n v_i \,x_i \quad\text{subject to}\quad \sum_{i=1}^n w_i \,x_i \;\le\; W. \] In other words, you may take any fraction (up to 1) of each item under the weight limit to maximize total value.

Input

A list of \(n\) items (item \(i\) has weight \(w_i\) and value \(v_i\)), and an integer knapsack capacity \(W\).

Output

The fractions \(x_i\in[0,1]\) for \(i=1,\dots,n\), and the maximum total value \[ V \;=\;\sum_{i=1}^n v_i\,x_i \] such that \(\sum_{i=1}^n w_i\,x_i \le W\).

Motivation/Applications

The fractional knapsack models scenarios where resources can be divided, such as loading cargo of divisible goods, investing funds across opportunities, or allocating bandwidth. Its greedy solution gives insight into resource allocation and serves as a foundation for more complex variants.

Examples

Example 1

Input:

ItemValueWeight
16010
210020
312030

Capacity \(W= 50\)

Output:

Take items 1, 2, and \(\tfrac{2}{3}\) of item 3 using all 50 of the weight.
Maximum value = \(240\).

Common Algorithms/Techniques

  • Greedy Algorithm that sorts items in decreasing order of value-to-weight ratio: \(O(n\log n)\).

Variants/Related Problems

  • 0-1 Knapsack (no fractions allowed)
