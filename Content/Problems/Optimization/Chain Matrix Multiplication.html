Chain Matrix Multiplication

Problem Description

Given a sequence of matrices \(A_1, A_2, \dots, A_n\), where each matrix \(A_i\) has dimensions \(p_{i-1} \times p_i\), determine the most efficient way to multiply the matrices together. That is, the goal is to fully parenthesize the product \[ A_1 A_2 \cdots A_n \] to minimize the total number of scalar multiplications.

Input

A sequence of integers \(p_0, p_1, \dots, p_n\) such that matrix \(A_i\) has dimensions \(p_{i-1} \times p_i\). The number of matrices is \(n\).

Output

The minimum number of scalar multiplications required to compute the matrix product, along with an optimal parenthesization (optional).

Motivation/Applications

Matrix chain multiplication arises in many applications involving linear algebra and optimization, such as graphics transformations, dynamic programming, and scientific computing. While matrix multiplication is associative, the order can greatly affect computational efficiency.

Examples

Example 1 (4 matrices)
Input: [40, 20, 30, 10, 30]
Matrices: \(A_1\ (40\times20)\), \(A_2\ (20\times30)\), \(A_3\ (30\times10)\), \(A_4\ (10\times30)\)
Output: 26000 scalar multiplications
Optimal parenthesization: \((A_1 \times (A_2 \times A_3)) \times A_4\)

Example 2 (6 matrices)
Input: [30, 35, 15, 5, 10, 20, 25]
Matrices: \(A_1\ (30\times35)\), \(A_2\ (35\times15)\), \(A_3\ (15\times5)\), \(A_4\ (5\times10)\), \(A_5\ (10\times20)\), \(A_6\ (20\times25)\)
Output: 15125 scalar multiplications
Optimal parenthesization: \((A_1 \times (A_2 \times A_3)) \times ((A_4 \times A_5) \times A_6)\)

Common Algorithms/Techniques

  • Dynamic Programming (Matrix Chain Order algorithm)
  • Recursive Divide-and-Conquer (inefficient without memoization)
