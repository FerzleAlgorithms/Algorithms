Merge

Problem Description

Given two lists that are each already sorted, the goal is to weave them together into a single list that remains sorted.

Input

Two sorted lists, List A and List B, containing elements drawn from the same ordered set. There may be duplicates.

Output

A single sorted list containing all elements of List A and List B, in non-decreasing order.

Motivation/Applications

Merging is the essential final step of Merge Sort, a powerful divide-and-conquer sorting method. It’s also used whenever you need to combine multiple sorted data streams (for example, merging log files by timestamp or merging database query results).

Examples

Example 1
Input:
List A = [1, 3, 5]
List B = [2, 4, 6]
Output:
[1, 2, 3, 4, 5, 6]

Example 2
Input:
List A = [1, 2, 7]
List B = [3, 5, 6]
Output:
[1, 2, 3, 5, 6, 7]

Common Algorithms/Techniques

  • Two-pointer (front pointers for each list)
  • Use of “sentinel” values to avoid checks at list ends
  • Copy the left half forward and the right half backward, then merge back by comparing elements starting from each end.

Variants/Related Problems

  • K-way merge (merging more than two sorted lists at once)
  • In-place array merge without extra memory
