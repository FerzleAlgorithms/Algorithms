Array Partition

Problem Description

Given an array A of n elements, select the first element p = A[0] as the pivot. Rearrange A in place so that all elements ≤ p precede it and all elements > p follow it, then place p into its correct position i. Return i, the final index of the pivot.

More generally (when not applied to an entire array), given an array A and indices low and high (with low < high), rearrange the elements in the portion of A from low through high as follows:

  1. Select a pivot element—typically the first element, A[low].
  2. Reorder the elements so that:
    • Every element less than or equal to the pivot is positioned to its left.
    • Every element greater than or equal to the pivot is positioned to its right.
    • The pivot element is positioned in its correct sorted location.
  3. The index of the pivot element is returned.

Input

An integer array A = [a0, a1, a2, ..., an-1], with n ≥ 1, and (optionally) a range (low,high), assumed to be (0,n-1) if not specified.

Output

The array A has been modified as specified, and the index i where p resides is returned.

Motivation/Applications

This partition step is central to QuickSort and Quickselect, facilitating divide-and-conquer sorting and selection. It also supports efficient order-statistic computations and data filtering.

Examples

Example 1
Input: A = [5, 3, 8, 4, 2, 7, 1, 6], p = 5
Possible Output: [3, 4, 2, 1, 5, 8, 7, 6]
Pivot index: 4

Note: Any solution is possible/valid as long as 1, 2, 3, and 4 occurs in some order at the beginning of the array, 5 occurs at index 4, and the 6, 7, and 8 occur in any order at the end of the array. Which one is returned will depend on the algorithm used.

Example 2
Input: A = [2, 2, 2, 2], p = 2
Possible Output: [2, 2, 2, 2]
Pivot index: 3

Note: Depending on implementation, any answer from 0 to 3 could be returned and would be valid.

Common Algorithms/Techniques

  • Lomuto's Partition: scans with a store index, moving elements ≤ pivot left and swapping pivot into place.
  • Hoare's Partition: uses two pointers that converge, swapping out-of-place elements then placing pivot.
  • In-Place Two-Pointer: generic approach with low/high indices.
