Sequential Search

Problem Statement

Given an array of elements and a value to search for, determine the index at which the value is located or return -1 if the value is not present in the array.

Algorithm Idea

Sequential search (also called linear search) checks each element in the array one-by-one, from the beginning until the desired value is found or the end is reached.

Talk Through Creation of Algorithm

The simplest way to find an item in a list is to examine each element sequentially. If the current element matches the searched value, return its index. If no match is found by the end of the list, return -1.

Pseudocode

SequentialSearch(array, value):
  for i from 0 to length(array) - 1:
    if array[i] == value:
      return i
  return -1

Interactive Demonstration

Time and Space Analysis

  • Time Complexity: O(n) - each element is checked once in the worst case.
  • Space Complexity: O(1) - no additional space needed.

Variations and Enhancements

  • Early exit if the array is sorted and the current element is greater than the searched value.
  • Parallel implementation to check multiple elements simultaneously.

Apply Ideas to New Problems (Reader Guided)

  • How would the algorithm change if the array were sorted?
  • Consider implementing sequential search on linked lists. What changes?

Reading Comprehension Questions

  1. What is the worst-case time complexity of sequential search? Why?
  2. If the search value is at the first position, how many comparisons are made?
Solutions
  1. O(n), because in the worst case it checks all elements once.
  2. Only one comparison is made.
